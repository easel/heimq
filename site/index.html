Kafka-compatible test broker

Run Kafka client tests without carrying a Kafka cluster.

heimq is a memory-first, single-node broker that speaks the Kafka wire protocol for local development, CI, and ephemeral workloads. Standard clients point at heimq unchanged; the project proves the supported surface with parity, benchmark, ecosystem, and conformance tests.

Two surfaces

The distribution is the heimq CLI: a fast, single-binary Kafka endpoint for test harnesses. The engine is the heimq-wire / heimq-broker crate family: wire scaffolding and broker semantics over pluggable backend traits.

North star: any Kafka client that runs against Redpanda should run against heimq for the in-scope semantic surface.

Use cases

Where heimq fits

  • Run service tests against a real Kafka wire surface without booting Kafka or Redpanda.
  • Keep CI feedback fast while retaining parity checks against a production-shaped broker.
  • Embed Kafka wire and broker semantics into Rust-native data systems through stable traits.

Boundaries

heimq is not a Kafka cluster replacement. It does not provide replication, controller quorum, or production Kafka durability. The distribution stays single-node; durability is a backend property for engine consumers.

Compatibility claims are bounded by the checked-in tests, fixtures, contracts, and benchmarks.
