Quick start

Start a Kafka-compatible endpoint with the least moving parts.

Build from source, run the published image, or install the chart. The smoke flow uses standard Kafka CLI tools, so the first check exercises the same client surface your tests use.

Source

Build and run from source

Use the release profile for a local binary, then point clients at port 9092.

cargo build -p heimq --release
./target/release/heimq --memory-only --host 0.0.0.0 --port 9092
Smoke test with standard Kafka CLI tools by producing to and consuming from a topic on localhost:9092. 
printf 'hello heimq\n' | kafka-console-producer.sh \
  --bootstrap-server localhost:9092 \
  --topic heimq-smoke

kafka-console-consumer.sh \
  --bootstrap-server localhost:9092 \
  --topic heimq-smoke \
  --from-beginning \
  --max-messages 1
Docker

Run the release image

The published image lives at ghcr.io/easel/heimq and uses the same host settings shown in the README. 

docker run --rm -p 9092:9092 \
  -e HEIMQ_HOST=0.0.0.0 \
  -e HEIMQ_MEMORY_ONLY=true \
  ghcr.io/easel/heimq:v0.1.0

Build and smoke-test the image locally before pushing tags.

just docker-build
./scripts/docker-image-smoke.sh heimq:local
Helm

Install the chart

The chart defaults are written so the image tag follows the release version published by the Docker workflow. 

helm install heimq charts/heimq \
  --set image.repository=ghcr.io/easel/heimq \
  --set image.tag=v0.1.0

Install the OCI chart published to GHCR.

helm install heimq oci://ghcr.io/easel/charts/heimq \
  --version 0.1.0
