Compatibility and testing

Compatibility is a measured surface, not a blanket promise.

heimq targets the Kafka flows used by standard clients, consumer groups, transactions, idempotent producers, benchmarks, and ecosystem tools. The supported surface is only as broad as the contracts and suites that exercise it.

Coverage

What is validated

  • Format, clippy, and workspace tests in CI.
  • Producer/consumer smoke flows through standard Kafka clients.
  • Differential parity workloads against Redpanda.
  • Benchmark smoke paths for Kafka perf-test and OpenMessaging Benchmark.
  • Ecosystem integration scripts for Kafka-speaking tools.
  • Container smoke validation for the release image.
  • Helm rendering and linting for the chart.

Useful commands

cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace --all-targets
just bench-smoke
just parity

Limits

Known constraints

  • Single-node only; no replication or controller quorum.
  • Memory-first operation; persistence is intentionally narrow.
  • Security protocols such as SASL and TLS are not enabled by default.
  • Kafka compatibility is bounded by the contracts, suites, and fixtures in this repository.
The README and this site are aligned on the same install commands, so a command that works here should work in the repository root as well.
