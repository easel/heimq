## Implementation Map

- **Topology**: single Cargo crate at `heimq/`. Modules: `protocol/` (codec + router), `handler/` (per-API handlers), `storage/` (LogBackend trait + memory + capabilities + dispatch + Postgres offsets behind `backend-postgres` feature), `consumer_group/` (coordinator + group + offset_store + backend trait), `server.rs`, `config.rs`, `error.rs`, `test_support.rs`.
- **Entry Points**: `main.rs` -> `server::run`; library reusable via `lib.rs`.
- **Public Interfaces**: Kafka TCP wire protocol (single listener, framed via `protocol::codec`).
- **Test Surfaces**: 145 unit/property tests in `src/**` (all green); `tests/contract.rs` 14 tests (1 red — see Gap Register); `tests/integration.rs` rdkafka tests (all `--ignored`-gated); `tests/postgres_offsets.rs` feature-gated.
- **Feature Flags**: `backend-postgres` (Cargo feature) gates Postgres OffsetStore.
- **Build/Deploy Surfaces**: `scripts/kcat-stress.sh`, `scripts/kcat-smoke.sh`, `scripts/compatibility-test.sh`, `scripts/stress-matrix.sh`.
- **Unplanned Areas**: none material. `compute_supported_apis` per-API gating is well-aligned with the design intent stated in heimq-2136e295's description.
- **Concern drift**: rust-cargo concerns implicit; CI test gate documented in test-plan section 6 is `cargo test --workspace --all-features` + `cargo llvm-cov --fail-under-lines 100 --fail-under-regions 100`. Not verified live in this review.
